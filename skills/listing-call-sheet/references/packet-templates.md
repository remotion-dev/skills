# Launch Content Plan + Three-Packet Templates

These are the output skeletons — but **the output itself is never a skeleton.** Fill every field with real, specific content. The only legitimate blanks are (a) clip IDs you assign, (b) genuinely unknowable post-sale data in the case-study shell, and (c) IDs/URLs the user must supply (GHL number, QR target) — mark those `[…TBD]`. Everything else is written out in full. A packet that reads like a form template has failed; it should read like a working document a real person picks up and acts on.

Keep clip IDs identical across all packets — they are the join key.

Plain-language rule (applies to every packet): write directions a non-jargon reader can act on. Lead with plain English, abbreviation in parentheses. "Medium shot (waist-up), camera slowly walks toward the door on a gimbal." Never "MS gimbal push." The shot glossary (`shot-glossary.md`) goes at the top of the videographer packet.

The launch slate comes from `references/launch-content-playbook.md` — read it before building PART 1.

## PART 1 — LAUNCH CONTENT PLAN

The strategy layer: the full set of posts + ads this one shoot powers, across the listing's 8 launch phases, plus the listing video itself. Each item has a one-line label so Graeham knows exactly what he's reading.

```
# LAUNCH CONTENT PLAN — <Address>
Price: $<x>  |  <beds>bd/<baths>ba  |  <sqft> sqft  |  <neighborhood>
Go-live: <date>   Shoot: <date>   Orientation: <portrait | landscape | both>
Avatar look: <chosen look>   Voice: ElevenLabs clone (default)
Launch keywords used: <e.g. BUY, TOUR, DOCS, OPEN, SELL — flag any new ones to create in GHL>
```

### A. The launch slate (every post + ad for this listing)

Instantiate the slate from `references/launch-content-playbook.md` for THIS property — every row, CORE flagged. Render it as the 8-phase table so nothing is forgotten:

```
PHASE 1 COMING SOON · PHASE 2 JUST LISTED · PHASE 3 DISCLOSURE · PHASE 4 LIFESTYLE
PHASE 5 OPEN HOUSE · PHASE 6 SOCIAL PROOF · PHASE 7 PRICE (if it stalls) · PHASE 8 SOLD
```

For each row: ★CORE? · asset · type · organic/paid · owner · footage source (clip IDs it reuses, or "copy only") · CTA + GHL keyword. CORE rows get ready copy in section C; the rest get a one-line brief in section D.

### B. The listing video (the spine)

Write it as TWO labeled lines, each self-explanatory:

```
LONG-FORM listing video  (<16:9 landscape | 9:16 portrait>)
• THE ARC (creative summary — NOT the words Graeham says): a 2–3 sentence description of how the
  finished video flows start to finish. The word-for-word script lives in the AGENT packet. Example:
  "Opens on Graeham at the curb, moves inside to the kitchen hero moment, out to the ADU, then the
  yard and solar, and closes on walkability with a text-keyword call to action."

SHORT-FORM cut  (<9:16 portrait>)
• A shorter edit built ENTIRELY from clips already filmed for the long-form — no extra shooting. List
  the exact clips it reuses BY CLIP ID, e.g.: "Reuses COOLEY-A01 (opener), COOLEY-L02 (island reveal),
  COOLEY-A04 (CTA)." Then one line on structure.
```

### C. CORE assets — ready to post (write these in full)

For each CORE asset in the slate (the ~13 must-dos), write the actual deliverable, not a brief:

```
<ASSET NAME>  (Phase N · <type> · <owner>)
• Hook:    the actual hook line.
• Body:    the caption (Hook → detail → CTA) OR the short script (for a Reel: 2-sec hook, 3 beats, CTA).
• CTA:     the exact wording incl. the GHL keyword.
• Footage: the clip IDs it reuses (or "static photos" / "graphic").
• Compliance: no price/rate/credit in video; price OK in caption/static; Fair Housing.
```

### D. The rest of the slate — one-line briefs

Each non-CORE asset gets a brief, fully written (angle, audience, the actual hook line, which already-filmed clip IDs it reuses, the GHL keyword). Don't pad — one tight line each.

### E. Case-study shell (fill AFTER the property sells)

Every post-sale field stays a placeholder until close of escrow. Do not publish until closed:
`[DOM TBD] · [SALE PRICE TBD] · [OVER/UNDER LIST % TBD] · [# OFFERS TBD] · [buyer-story angle TBD]`.

## PART 2 — VIDEOGRAPHER PACKET (numbered shot list)

Open with the shot glossary (`shot-glossary.md`), then the three categories. **Plan the listing shots to cover the WHOLE launch's footage needs**, not just the long-form video — see the footage-coverage rule in `references/launch-content-playbook.md`. At minimum the listing shots must yield: the 4-photo status set (front exterior / strong interior / alternate exterior / rear exterior), reveal/hero cuts, a full room-by-room walkthrough, neighborhood B-roll, and a few open-house-able clips.

Every field-capture shot carries **seven** things (composition is the field crew's must-have):

1. **Clip ID** · 2. **What's in frame** (specific) · 3. **Composition** (how much of the subject, camera height, sky-vs-ground ratio, exactly where to stand) · 4. **Camera move** (plain English) · 5. **Shot size** (plain English) · 6. **Location & light** · 7. **Hold** (seconds).

```
# VIDEOGRAPHER PACKET — <Address>   Shoot: <date>
For: <videographer name/email>
Orientation: <…>   Export each clip as <CLIPID>_<orientation>.<ext>. Nothing else in the filename.

### SHOT GLOSSARY  (paste shot-glossary.md here so directions are readable cold)

NOTE: Category (c) avatar-source shots are STRICTER than B-roll — locked tripod, even light, held
framing, 2+ minutes (see the callout). The rest can be filmed loose.
```

### (a) LISTING SHOTS — for this property's video AND the launch slate

Write each with full composition. This is the level of detail "done" looks like:

```
COOLEY-L01 — Front elevation  (also = the "Just Listed" status photo)
• In frame: the full front of the house with the classic red/white FOR SALE sign on the lawn.
• Composition: capture the house corner-to-corner; leave ~15–20% sky above the roofline so the roof
  isn't clipped, house through the middle, front lawn + sign filling the bottom third. Stand at the
  end of the front walk, dead center on the house. Camera at chest height.
• Camera move: start static for 2s, then a slow push-in (camera eases straight toward the door).
• Shot size: wide shot (whole house).
• Location: public sidewalk centered on the front walk.
• Light: golden hour, sun behind the camera so the front face is lit (no backlight, no hard shadows).
• Hold: 6–8s.
```

Mark which launch asset each listing shot feeds in a short tag, e.g. `→ Reveal Reel + Hero Carousel`, `→ status photo (Sold)`, `→ Neighborhood Walkthrough`. The **4-photo status set** must be explicitly present: front exterior (Just Listed), a strong interior (Open House), an alternate exterior angle (In Contract), and a rear exterior or second interior (Sold).

### (b) EXTRA FOOTAGE TO BANK — for your OTHER upcoming videos ("demand-backed B-roll")

Lead with the plain-English explainer: while the videographer is already on location, grab a few extra clips for OTHER videos Graeham already has planned, so the trip does double duty. "Demand-backed" = only footage a real planned video actually needs (never random spares). Cap **5 scenes**; each names the upcoming video it feeds. These do NOT go in this listing's video.

```
COOLEY-B01 — Peninsula street-level establishing
• What to film: a slow gimbal walk down the block, neutral homes and trees, no people in focus.
• Composition: eye-level, lead with the sidewalk; keep it generic (no house identifiable).
• Feeds queued video: "AB 1482 rent-cap explainer" — needs real Peninsula street B-roll.
```

### (c) AVATAR-SOURCE SHOTS — only if a new look is needed

Capture only if a CORE talking-head asset (e.g. the Disclosure Drop narration) needs a HeyGen look that isn't already trained. Stricter than B-roll: locked tripod, even frontal light, held framing, **2–3 minutes** per look (see `avatar-source-specs.md`). If the look already exists, skip and mark the line `AVATAR (HEYGEN — existing look)`.

```
COOLEY-A01 — Digital-twin source (only if training a new look)
• Composition: medium close-up (chest-up), straight-on, eye-level, tripod-locked, Graeham centered,
  consistent headroom. Open shade or soft frontal light, no backlight.
• Hold: 2–3 minutes of continuous delivery (gives HeyGen enough source).
```

## PART 3 — AGENT (GRAEHAM) PACKET

What Graeham needs: his exact words, and the rollout he's driving.

```
# YOUR PACKET — <Address>
```

### The listing script (every line, in order)

One block per spoken line — the ACTUAL words, never a summary. Tag each line **Live to camera** or **Voiceover**, with the clip ID and what to do while saying it.

```
COOLEY-L01 · Live to camera · at the curb
"<exact words Graeham says>"
(what to do while saying it)

COOLEY-L13 · Live to camera · YOUR CALL TO ACTION
"<exact CTA wording, incl. the GHL keyword>"
(look into the lens, warm and direct)
```

Talking points: features / price / lot / commute / market only — never who lives in the area (Fair Housing).

### Editor rules (standing)

- **No DRE number anywhere on screen.** **No price / rate / credit in any video** (voiceover or on-screen graphic). Address is fine.
- Graphics are built on green screen, keyed in CapCut (`watts-motion-graphics`).
- B-roll = voiceover or music only, never two competing voices. Hero beat = the named clip (music drop). All voice = ElevenLabs clone.
- Render every asset in its required aspect (16:9 / 1:1 / 9:16). Prepend the geo-tagged thumbnail to the 1:1 and 9:16 files.
- **AI b-roll (only if needed):** the `AI` clip IDs with their prompts (cinematic-hooks → higgsfield-video).
- **Bank clips — do NOT use in this listing's content:** the `B` clip IDs, each filed to its queued video.
