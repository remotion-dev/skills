# Avatar-Source Capture Specs

Avatar-source shots (`A` category) are footage of Graeham filmed specifically to **train or drive a HeyGen avatar** — not necessarily to appear in the cut as-is. They have a **stricter capture bar than B-roll**, for a technical reason:

> HeyGen builds the avatar from the source clip. If the source is shaky, unevenly lit, or loosely framed, the avatar inherits those problems and looks wrong. B-roll can be run-and-gun; avatar source cannot. It has to be **stable, evenly lit, and consistently framed.**

## First decide: do we even need to capture avatar source on this trip?

- **If the chosen look is already trained** (e.g. `digital_twin` already exists in HeyGen), you do **NOT** need new avatar-source footage to use it. The avatar can already say any script via the voice clone. In that case, mark the relevant lines `AVATAR (HEYGEN — existing look)` and skip `A` field capture entirely.
- **Capture avatar source only when** (a) Graeham wants to train a *new* look from this trip, or (b) he wants fresh on-location source (e.g. delivering lines on this specific street) for a new look. If neither is true, the videographer's `A` workload is zero — say so plainly rather than inventing avatar shots.

Don't pad the shoot with avatar-source clips nobody asked for. If the listing video is being driven by an existing trained avatar, the only on-camera Graeham footage you might still want is **real live talking-head** (which is a `REAL-INT`/`REAL-EXT` shot of him actually speaking, used directly) — that's a different thing from avatar source and is fine to film loosely.

## HeyGen's real requirements (verified — use these numbers)

These are HeyGen's published guidelines, not a guess. Match the avatar type Graeham is creating:

| Avatar type | Source footage needed | Notes |
|---|---|---|
| **Digital Twin (Avatar IV)** — the high-fidelity look | **≥ 2 minutes, ideally 3–5 minutes** per look | 30 seconds is the bare-minimum quick-start, not the target. More footage = better twin. |
| **Avatar 5 / Instant Avatar** — fast lightweight clone | as little as **15 seconds** | phone/webcam grade; lower fidelity |

**Technical bar (both):** ≥ 1080p, 30fps (4K/60fps better). Record in 16:9 or 9:16 (capture for whatever Step-0 orientation was chosen; "both" → shoot 4K and crop). **Only Graeham's voice on the recording — no other speakers, no crew talking.** He can move or even walk, but for a consistent twin he should **stay centered and controlled.**

So when you spec an `A` shot, the default hold is **2–3 minutes of clean source per look** (not 60 seconds). If Graeham only wants a quick Avatar-5-grade clip, 15–30s is acceptable — but say which avatar type it's for so he captures enough.
