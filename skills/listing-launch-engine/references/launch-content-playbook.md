# Launch Content Playbook — the content blueprint

This is the **content-plan backbone** of the Call Sheet. When a new listing comes in, this is the full slate of posts and ads to produce across the listing's life, organized by phase. The Call Sheet instantiates this slate for the specific property, and — critically — the videographer shot list is planned so **one trip captures the footage the whole launch needs**, not just the single listing video.

## How the Call Sheet uses this

1. **Build the per-listing launch slate** — list every asset below for THIS property, CORE ones flagged. Nothing gets forgotten.
2. **Produce now vs brief** — write ready copy/scripts for the **CORE** assets (the ~13 must-dos); write a one-line brief for the rest (angle + hook + reused clip IDs + CTA). The same depth rule as the rest of the Call Sheet.
3. **Feed the shot list** — every asset whose **Footage** column is a clip category becomes a requirement on the videographer packet. Decompose so the listing shoot (category `L`) yields all of it (see "Footage coverage rule").
4. **Schedule** — the launch slate becomes the AGENT packet's posting calendar (phase → timing → asset → CTA/keyword) and the EDITOR packet's derivative render list.

## Footage coverage rule (the whole point of the merge)

The one trip must capture enough listing footage (`L` clips + listing photos) to feed every footage-dependent asset. At minimum, plan listing shots that yield:

- **The 4-photo status set** — four distinct hero stills/clips so the same listing can post four ways across its life: **front exterior** (Just Listed), a **strong interior** (Open House), an **alternate exterior angle** (In Contract / Pending), and a **rear exterior or second interior** (Sold). This is what makes one listing look like four deals.
- **Reveal / hero cuts** — for the Reveal Reel and the Hero Carousel.
- **Full walkthrough coverage** — every room + flow, for the YouTube tour and the POV walkthrough.
- **Neighborhood B-roll** — exteriors, street, parks/transit/amenities (Fair Housing: places and amenities, never who lives there) for the Neighborhood Walkthrough and lifestyle posts.
- **Open-house-able clips** — a few energetic walk-up / interior clips usable for the open-house Reel and Story.
- **Avatar / talking-head source** — only if a CORE talking-head asset (Disclosure Drop, narration) needs a look that isn't already trained (see `avatar-source-specs.md`).

If an asset's footage need isn't on the shot list, it can't be produced — so decompose from this slate, not just the one script.

`TOUR`, `DOCS`, `OPEN` are new for listing plays — either create the ManyChat/GHL comment-to-DM automations or map them onto an existing keyword. Flag this to Graeham on any run that uses them.

## Operational specs (the settings John uses)

- **Render aspect ratios:** 16:9 (YouTube), 1:1 (FB/IG feed), 9:16 (Reels/Stories/Shorts). Peter renders all three from one shoot.
- **Meta Lead Ads:** Housing Special Ad Category · 15-mile radius around the listing · "More Volume" instant form · prefill First/Last/Email/Phone (delete Full Name) · timeline question (0-3 / 3-6 / 6+) · privacy → graehamwatts.com/site/privacy-terms · push to GoHighLevel via Make.com/ActivePieces · drive to Graeham's OWN IDX, never Zillow/Redfin.
- **Curiosity-gap creative:** withhold kitchen / primary bed / primary ensuite so viewers must DM or click to see them.
- **IG geo-fence:** stand in the neighborhood, boost → "More website visits" → Local audience saved as "The Neighborhood."
- **Geo-tagged thumbnails:** geoimgr.com with Graeham's Maps lat/long in EXIF, prepended to the 1:1 and 9:16 files for the cover.
- **Retargeting:** GHL → Meta Custom Audience (60-day website retention); the Sold ad retargets the Just-Listed ad viewers. Mint tracked links via the `switchy-engine` skill.
- **Scheduling:** VistaSocial or the GHL Social Planner. Group 1:1 for feeds, 9:16 for Reels/Stories.

## Variation-prompt template (for generating any asset's copy)

When the team wants more copy for an asset, this is the prompt shape (the standalone dashboard's "Copy variation prompt" buttons produce it filled). The humanizer rules are non-negotiable.

```
You are creating [ASSET] for a new listing, as Graeham Watts, REALTOR at Intero Real Estate,
DRE# 01466876. Market: East Palo Alto / Peninsula.
LISTING: address ___ | beds/baths/sqft ___ | price ___ | 3 features ___ | open house ___ | neighborhood ___.
ASSET: [name] (Phase N). Platform/format/aspect/goal/owner. CTA: [cta] (GHL keyword: [kw]).
DELIVERABLE: [format spec].
COMPLIANCE:
- No price/rate/credit in any VIDEO (VO or on-screen). Address is fine. Price OK in captions/static slides.
- Fair Housing: features, price, market, amenities, commute only. Never who lives there; no school rankings.
- DRE# 01466876 on paid ads.
HUMANIZER: no em dashes; no "stands as a testament / vibrant / nestled / must-see"; name real
sources; no fake-depth -ing tails; no forced rule-of-three; "is/are/has" not "serves as"; straight
quotes; vary sentence length; first person when it fits; concrete numbers over abstractions; if a
line sounds like a press release, rewrite it.
Produce 3 variations.
```

## Compliance (non-negotiable on every asset)

- **No price / rate / credit inside any VIDEO** — voiceover or on-screen graphic. Street address is fine. Price lives in captions and static carousel slides only. ("Sold over asking" is fine in video; the dollar figure is not.)
- **Fair Housing** — property features, price, market data, amenities, commute only. Never describe who lives in an area; no "safe / good / family-friendly"; no school rankings as a selling point.
- **DRE# 01466876** on every paid ad. **No DRE number on screen in video graphics.**
- **Private** — seller price-reduction talking points are agent-to-seller only.
