## 动画原则

### 1. Anticipation (预备动作)

动作前先反向蓄力：

```json
"s": {"a":1,"k":[
  {"t":0, "s":[100,100]},
  {"t":8, "s":[95,105]},      // 蓄力下蹲 (反向)
  {"t":25,"s":[115,90]},      // 主动作 (爆发)
  {"t":40,"s":[100,100]}      // 恢复
]}
```

### 2. Squash & Stretch (压扁拉伸)

**核心规则**: `X × Y ≈ 10000` 保持体积守恒

```json
{"t":0, "s":[100,100]},       // 10000 (原始)
{"t":10,"s":[120,83]},        // 9960 ≈ 守恒 (压扁)
{"t":20,"s":[85,118]}         // 10030 ≈ 守恒 (拉伸)
```

应用场景：
- 起跳蓄力 → 压扁 `[110,91]`
- 腾空 → 拉伸 `[85,118]`
- 落地 → 压扁 `[125,80]`
- 恢复 → `[100,100]`

### 3. Follow-Through (跟随过冲)

到达目标后过冲再回弹：

```json
"r": {"a":1,"k":[
  {"t":0, "s":[0]},
  {"t":15,"s":[95]},           // 过冲目标(90°)
  {"t":25,"s":[88]},           // 回弹
  {"t":35,"s":[90]}            // 最终值
]}
```

### 4. Staggered Timing (交错时间)

多元素按延迟依次入场：

```json
{"ind":1,"st":0,"ip":0},      // 立即
{"ind":2,"st":3,"ip":3},      // 3帧延迟
{"ind":3,"st":6,"ip":6},      // 6帧延迟
{"ind":4,"st":9,"ip":9}       // 9帧延迟
```

### 5. Secondary Action (次要动作)

主动作引发的附带效果。例：人物落地 → 指针振荡

```json
// 指针阻尼振荡 (落地帧=40)
"r": {"a":1,"k":[
  {"t":40,"s":[0]},
  {"t":41,"s":[35]},    // 瞬间偏转
  {"t":51,"s":[-25]},   // 反弹 (振幅递减)
  {"t":58,"s":[12]},
  {"t":63,"s":[-5]},
  {"t":66,"s":[0]}      // 归零
]}
```

## Layer Parenting 骨骼层级

用 `parent` 属性创建父子关系。移动父层 → 所有子层跟随。

```json
{"layers":[
  {"ind":14,"nm":"Shadow","ks":{"p":{"a":0,"k":[340,195,0]}}},
  {"ind":1,"nm":"Head","parent":14,"ks":{"p":{"a":0,"k":[88,-84,0]}}},
  {"ind":2,"nm":"Body","parent":14,"ks":{"p":{"a":0,"k":[0,-50,0]}}}
]}
```

**关键**: 子层位置是**相对于父层**的坐标。

**骨骼策略**:
- **Shadow 做父级**: 移动阴影 = 移动整个角色
- **Body 做父级**: 转身 = 头和四肢跟随
- **Upper Arm 做父级**: 肩膀旋转 = 前臂和手跟随

```
角色骨骼树:
Shadow (整体移动)
├── Head
├── Body
├── UpperArm_L
│   └── Forearm_L
│       └── Hand_L
└── UpperArm_R
    └── Forearm_R
        └── Hand_R
```

## Frame-by-Frame 逐帧动画

复杂角色动画（走路/跑步循环）的最专业技术。创建多个"姿态"，通过 `ip/op` 依次显隐。

```json
{"layers":[
  {"nm":"Pose1","ip":0,"op":6,"shapes":[/* 姿态1 */]},
  {"nm":"Pose2","ip":6,"op":12,"shapes":[/* 姿态2 */]},
  {"nm":"Pose3","ip":12,"op":18,"shapes":[/* 姿态3 */]},
  {"nm":"Pose4","ip":18,"op":24,"shapes":[/* 姿态4 */]}
]}
```

**计算**: `总帧数 = 姿态数 × 每姿态帧数`, `时长 = 总帧数 / 帧率`

**优势**:
- 每个姿态可以有完全不同的形状（不需要相同顶点数）
- 比 morphing 更自然
- 用于: 走路循环、跑步、复杂形变

## 形状修饰器

### Trim Path (路径裁剪/绘制)

描边动画的核心。控制路径的 start/end 百分比：

```json
{"ty":"tm",
 "s":{"a":0,"k":0},
 "e":{"a":1,"k":[
   {"t":0,"s":[0],"o":{"x":[0.33],"y":[0]},"i":{"x":[0.67],"y":[1]}},
   {"t":45,"s":[100]}
 ]},
 "o":{"a":0,"k":0},"m":1}
```

### Repeater (重复)

```json
{"ty":"rp","c":{"a":0,"k":8},
 "tr":{"r":{"a":0,"k":45},"so":{"a":0,"k":100},"eo":{"a":0,"k":30}}}
```

### Offset Path (路径偏移)

```json
{"ty":"op","a":{"a":1,"k":[{"t":0,"s":[0]},{"t":30,"s":[8]}]},"lj":2}
```

## Stroke+Fill 轮廓风格

专业角色动画使用描边+填充组合：

```json
{"ty":"gr","it":[
  {"ty":"sh","ks":{...}},
  {"ty":"st","c":{"a":0,"k":[0.259,0.153,0.141,1]},"w":{"a":0,"k":1},"lc":2,"lj":2},
  {"ty":"fl","c":{"a":0,"k":[0.302,0.604,0.816,1]}},
  {"ty":"tr",...}
]}
```

| 属性 | 值 | 说明 |
|------|-----|------|
| `lc` | 1=Butt, 2=Round, 3=Square | 线端样式 |
| `lj` | 1=Miter, 2=Round, 3=Bevel | 连接样式 |

## 常用动画配方

### Pop-In 弹入 (Bounce)
```json
"s":{"a":1,"k":[
  {"t":0,"s":[0,0],"o":{"x":[0.34],"y":[1.56]},"i":{"x":[0.64],"y":[1]}},
  {"t":18,"s":[110,110],"o":{"x":[0.33],"y":[0]},"i":{"x":[0.67],"y":[1]}},
  {"t":30,"s":[100,100]}
]},
"o":{"a":1,"k":[{"t":0,"s":[0]},{"t":12,"s":[100]}]}
```

### 心跳 (Lub-Dub)
```json
"s":{"a":1,"k":[
  {"t":0,"s":[100,100]}, {"t":8,"s":[115,115]},
  {"t":12,"s":[90,90]},  {"t":18,"s":[105,105]},
  {"t":45,"s":[100,100]}
]}
```

### 弹跳球 (含 Squash & Stretch)
```json
"p":{"a":1,"k":[
  {"t":0,"s":[100,50],"o":{"x":[0.55],"y":[0.055]},"i":{"x":[0.675],"y":[0.19]}},
  {"t":20,"s":[100,250]},  // 落地
  {"t":40,"s":[100,100]},  // 弹起
  {"t":60,"s":[100,250]}   // 再落
]},
"s":{"a":1,"k":[
  {"t":18,"s":[90,110]},   // 下落拉伸
  {"t":20,"s":[120,80]},   // 落地压扁
  {"t":28,"s":[100,100]}   // 恢复
]}
```

### 连续旋转 (Spinner)
```json
"r":{"a":1,"k":[{"t":0,"s":[0]},{"t":120,"s":[360]}]}
```
配合 Trim Path 实现加载动画。

## 专业时间控制

| 类型 | FPS | 帧数 | 时长 | 用途 |
|------|-----|------|------|------|
| 快速循环 | 60 | 36 | 0.6s | 跑步循环 |
| 标准循环 | 30 | 24 | 0.8s | 走路循环 |
| 慢循环 | 30 | 60 | 2.0s | 待机呼吸 |
| 过渡 | 60 | 45 | 0.75s | 入场 |

**循环完美闭合**: 最后帧 `op` 不渲染，只标记循环点。确保首帧=末帧附近状态。

**图层顺序 = Z深度**: 数组中靠前的图层渲染在上方。

## 常见错误排查

| 错误 | 原因 | 解决 |
|------|------|------|
| 不循环 | 首末帧值不匹配 | 匹配首末关键帧 |
| 僵硬 | 无缓动 | 添加贝塞尔 i/o |
| 卡顿 | 关键帧太稀疏 | 添加中间帧 |
| 形变异常 | 顶点数不同 | morph 必须相同顶点数 |
| 旋转中心错 | 锚点不对 | 设置 `a` 到旋转中心 |
| 角色动画丑 | 用 morph 做复杂形变 | 改用 Frame-by-Frame |
| 画面静止 | 中间帧缺 i/o 或 h | 确保每个中间帧有缓动或 `h:1` |

## 性能指南

1. 图层数 < 15
2. 每个形状顶点 < 20
3. 30fps 通常够用，60fps 用于超流畅
4. 避免嵌套 Repeater
5. 图标 < 50KB, 复杂动画 < 200KB
