# 输入到输出映射示例

## 概述

本文档展示一个真实案例：从输入的静态 SVG + Lottie 结构 JSON 映射到输出的动画 Lottie JSON。

## 输入文件

### SVG 文件描述

一个 375×360 的插画，包含三组视觉元素：
- **卷尺/尺子**：左侧区域，由椭圆底座、竖线刻度、矩形手柄等组成
- **称/天平**：中间区域，由指针、罗盘和手机外壳形状组成
- **人物**：右侧区域，由头（圆形）、身体（矩形+圆角）、四肢（描边线条）组成

### 输入 Lottie JSON 结构要点

```
顶层:
  fr: 60, ip: 0, op: 300 (5秒@60fps)
  w: 750, h: 720 (SVG 尺寸×2)

assets (预合成):
  comp_0 "称"   → 包含指针、圆点、罗盘、手机壳组等图层
  comp_1        → 手机壳前景形状
  comp_2        → 手机壳背景形状  
  comp_3 "人"   → 包含头、身体、左手、右手、左腿、右腿图层
  comp_4 "尺"   → 包含各刻度线、底座形状、把手等图层
  comp_5        → 底座椭圆细节

layers (主合成):
  ind:1 ty:0 "称"  → refId:comp_0, 位置 [390,476]
  ind:2 ty:0 "人"  → refId:comp_3, 位置 [542,476]  
  ind:3 ty:0 "尺"  → refId:comp_4, 位置 [213.567,487]
  ind:4 ty:4 "背景" → 375×360 纯色矩形
```

**关键特征**：所有变换属性 `a: 0`，表示完全静态，无任何动画。

## 输出文件

### 动画需求

创建一个"计量计算"主题动画：小人沿着卷尺和称之间的路径行走/跳跃。

### 输出 Lottie JSON 结构要点

```
顶层:
  fr: 25, ip: 0, op: 90 (3.6秒@25fps)
  w: 750, h: 720
  nm: "计算动画"

assets:
  image_0 → 称的 PNG base64 (128×128)
  image_1 → 卷尺的 PNG base64 (160×128)

layers:
  ind:1 "遮罩-卷尺"  td:1  → 矩形+椭圆遮罩形状（静态）
  ind:2 "人-卷尺"    tt:1  → 人物动画（核心动画层！）
  ind:3 "遮罩-称"    td:1  → 矩形遮罩形状（静态）
  ind:4 "称指针"     tt:1  → 指针旋转动画
  ind:5 "背景矩形"          → 白色背景
  ind:6 "weight.png" ty:2  → 称的图片
  ind:7 "卷尺.png"   ty:2  → 卷尺图片
  ind:8 "纯色背景"   ty:1  → 浅蓝背景色
```

### 核心动画变化对比

#### 1. 元素处理方式变化

| 输入 | 输出 | 原因 |
|------|------|------|
| 矢量预合成 (ty:0 → comp_0) | PNG 图片 (ty:2) | 简化渲染，元素无需形变 |
| 矢量预合成 (ty:0 → comp_4) | PNG 图片 (ty:2) | 简化渲染 |
| 矢量预合成 (ty:0 → comp_3) | 手动路径动画 (ty:4) | 需要复杂路径形变动画 |

#### 2. 人物行走动画（关键帧路径）

人物的行走通过三组动画实现：

**A. 手臂/腿部摆动 → 路径关键帧动画**

手臂动画使用路径 `sh` 关键帧，在不同帧定义不同的贝塞尔路径形状：

```
帧 0:  双臂自然下垂姿态
帧 4:  双臂张开（起跳）
帧 7:  双臂半举
帧 10: 双臂收拢
帧 13-16: 着地姿态 + 行走起步
帧 21-29: 行走中摆臂
帧 31-39: 跳跃飞越
帧 41-52: 着陆 + 继续行走
帧 56-85: 到达目的地 + 待机摆动
```

**B. 身体位置 → 位置关键帧**

```json
"p": {
  "a": 1,
  "k": [
    {"t": 4,  "s": [197.5, 523.5, 0]},  // 起始位置（画面外下方）
    {"t": 16, "s": [197.5, 408.5, 0]},  // 跳上卷尺
    {"t": 49, "s": [197.5, 408.5, 0]},  // 保持
    {"t": 52, "s": [197.5, 408.5, 0]}   // 保持
  ]
}
```

**C. 头部跟随 → 椭圆位置关键帧**

头部椭圆的位置跟随身体运动：

```
帧 0:  [-3, -8]   (相对身体偏移)
帧 4:  [-3, -12]  (起跳时头部上移)
帧 21: [2, -20]   (行走中头部前探)
帧 39: [151, -37] (飞越中大幅偏移)
帧 56: [323, 31]  (到达目的地)
```

#### 3. 称指针旋转动画

```json
"r": {
  "a": 1,
  "k": [
    {"t": 40, "s": [0]},     // 人物落地前静止
    {"t": 41, "s": [44]},    // 人物落地，指针快速偏转
    {"t": 51, "s": [-52]},   // 反弹
    {"t": 55, "s": [31]},    // 回摆
    {"t": 58, "s": [-28]},   // 再次反弹（幅度递减）
    {"t": 60, "s": [15]},
    {"t": 62, "s": [-7]},
    {"t": 63, "s": [0]}      // 最终归位
  ]
}
```

这是一个经典的**阻尼振荡**效果：振幅逐渐递减直到归零。

#### 4. 遮罩使用

遮罩用于限制人物动画的可见区域：

```
遮罩定义层 (td:1):
  矩形1: 覆盖卷尺区域
  矩形2: 扩展可见区域
  椭圆:  底部圆角过渡

被遮罩层 (tt:1):
  人物动画图层
```

这样人物在卷尺之外的区域会被裁切，实现"在卷尺上行走"的视觉效果。

## 关键映射规则总结

1. **不需要形变的复杂元素** → 转换为 PNG base64 嵌入 `assets`，用 `ty:2` 图片图层引用
2. **需要复杂动画的元素** → 拆解为基本形状，用 `ty:4` 形状图层 + 路径关键帧
3. **遮罩效果** → 用 `td:1` + `tt:1` 的相邻图层对实现
4. **弹跳/物理效果** → 使用阻尼振荡的关键帧序列（振幅递减）
5. **行走动画** → 分解为身体位置、肢体路径形变、头部跟随三组独立动画
6. **帧率调整** → 可以从设计稿的 60fps 调整为 25fps 减小文件体积
