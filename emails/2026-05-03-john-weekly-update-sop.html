Monday Weekly Seller Updates — How To Run Them

A 15-minute Monday checklist for John · Generates branded weekly status reports for every active listing

Hey John — every Monday afternoon, sellers expect a polished weekly update on where their listing stands. Cowork generates the actual report — your job is to feed it the data, run the skill, and forward the result to Graeham for review. Should take you ~15 minutes total per listing.

This SOP is the same every week. You'll get a reminder email at 12:30 PM Monday with the active listings list and a link back to this page.

How the day flows: 12:30 PM email reminds you → you pull data + drop in Cowork → Cowork drives Chrome to grab live Zillow/Redfin numbers → Cowork drafts the seller email + sends it to Graeham for review → Graeham approves → email goes to seller.

⏱
Total time per listing: ~15 minutes · Total weekly time: ~15 min × number of active listings
Step 1 of 5 ~5 min

Pull the showing data from ShowingTime / Supra

For each active listing, log into our showing platform and export the agent activity report.

How To

1. Go to showingtime.com (or whichever platform Graeham uses for that listing)

2. Find the active listing → Reports → Agent Activity Report

3. Set date range to "Since list date"

4. Export to Excel (.xlsx)

5. Save with the filename pattern: [address-keyword]-showing-[YYYY-MM-DD].xlsx
Example: cooley-showing-2026-05-04.xlsx

⚠
If the showing platform also exports the disclosure pulls (Glide), grab that in the same file or as a second sheet. Otherwise see Step 2.
Step 2 of 5 ~3 min

Pull the disclosure data from Glide

Only needed if the disclosure pulls weren't included in the ShowingTime export.

How To

1. Log into Glide (glide.com) using Graeham's listing-package credentials

2. Find the property's disclosure package → Activity tab

3. Export the agent view/download log to Excel

4. Save with the same filename pattern: [address-keyword]-glide-[YYYY-MM-DD].xlsx

Step 3 of 5 ~2 min

Open Cowork and drop the files

Open your Cowork app → start a new chat. Drag the showing file (and disclosure file, if separate) into the chat. Wait for the upload to finish.

✓
Quality check: The uploaded files should appear at the top of the chat. If you don't see them, retry the upload before moving to Step 4.
Step 4 of 5 ~3 min

Run the weekly-listing-update skill

Type this exact prompt into the Cowork chat (substitute the property address):

Prompt to type

Run the weekly-listing-update skill for [PROPERTY ADDRESS]. Use the showing data I just uploaded. Generate the report and push to GitHub Pages.

Cowork will:

  • Parse the showing & disclosure data
  • Pull current market context (EPA average DOM, median PPSF)
  • Open Claude in Chrome and navigate to Zillow, Redfin, Homes.com to grab live view counts and saves
  • Generate the branded HTML status report
  • Push it to GitHub Pages → return a permanent hosted URL
  • Create a Gmail draft to the seller — but won't send it

Total runtime: about 90–120 seconds (Chrome scraping adds ~30 seconds). You may see browser tabs open and close during this — that's normal.

⚠
Chrome permission: The first time you run this on a new computer, Chrome will ask you to approve navigation to Zillow/Redfin/Homes.com. Click Allow — the approval is remembered.
Step 5 of 5 ~2 min

Forward to Graeham for review

Cowork will return a hosted URL like:
https://graehamwatts.github.io/skills/emails/2026-05-04-1908-cooley-weekly-status.html

Open the URL to spot-check it looks right (numbers match the data, no obvious typos), then send Graeham a quick message:

Message to Graeham (paste / adapt)

Weekly update ready for [Address]. Hosted: [URL]. Gmail draft is in your inbox to [Seller Email]. Status: [emoji] [or whatever the report shows]. Decision needed: [yes/no, on what]. Let me know if you want any edits before I/you send.

Graeham reviews → either approves (you/he forwards the Gmail draft) or sends back edits. Repeat for any other listings.

Red flags · stop and message Graeham

Don't proceed and send if you see any of these:

  • Showing count drops dramatically vs. last week with no obvious reason
  • A new offer or price change appears in the data that Graeham didn't tell you about
  • The hosted URL doesn't load, or the formatting looks broken
  • Sensitive seller info accidentally appears (financial, personal) — these reports are seller-facing only
  • Cowork errors out, hangs, or returns "skill not found"
